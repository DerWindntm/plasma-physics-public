1. PIC 的步骤说明 1. PIC Algorithm Steps 1. PIC アルゴリズムの手順

PIC 方法简介 Introduction to PIC PIC 法の概要

质点网格法（PIC）是目前用于模拟等离子体动态行为最广泛使用的方法之一。作为一种成熟且确立已久的方法，现代 PIC 代码能够高效地扩展到目前最大规模的超级计算机上运行，而这种大规模计算能力往往是模拟真实世界现象所需足够分辨率的前提条件。 The Particle-In-Cell (PIC) method is one of the most widely used approaches to simulating plasma dynamics. As a mature and well-established method, modern PIC codes can efficiently scale to the largest supercomputers available, which is often required to provide sufficient resolution for real-world phenomena. 粒子セルイン法（PIC 法）は、プラズマ動態のシミュレーションに最も広く使用されている手法の一つです。成熟した確立済みの手法として、現代の PIC コードは現存する最大規模のスーパーコンピュータへ効率的にスケールでき、現実世界の現象を十分な解像度でシミュレートするために必要とされています。

PIC 代码的核心包含两个相互耦合的求解器：粒子推进器（particle pusher）和场求解器（field solver）。两者通过插值操作交替工作，在离散时间步中不断更新超粒子的位置、动量，以及网格上的电场和磁场。 At its core, a PIC code contains two coupled solvers: the particle pusher and the field solver. They work alternately via interpolation, updating superparticle positions and momenta together with electric and magnetic fields on the grid at each discrete time step. PIC コードの核心は、粒子プッシャーと場ソルバーという二つの連成ソルバーです。この二つは補間操作を介して交互に動作し、離散的な時間ステップごとに超粒子の位置・運動量と、グリッド上の電磁場を更新します。

超粒子与网格 Superparticles & Grid 超粒子とグリッド

等离子体粒子被建模为所谓的超粒子（superparticle），每个超粒子代表大量真实粒子。例如，一个超粒子可能代表 \(10^5\) 个电子。超粒子存在于连续的相空间中，其集体行为在统计意义上代表了对应真实等离子体系统的行为。电场和磁场则定义在一个离散网格上。 Plasma particles are modelled as superparticles, each representing a large number of real particles — for example, \(10^5\) electrons. Superparticles exist in continuous phase space, and their collective behaviour statistically represents the real plasma. Electric and magnetic fields are defined on a discrete grid. プラズマ粒子は超粒子（スーパーパーティクル）としてモデル化され、各超粒子は多数の実粒子（例：電子 \(10^5\) 個）を表します。超粒子は連続的な位相空間に存在し、その集合的挙動が実際のプラズマ系を統計的に表現します。電磁場は離散グリッド上で定義されます。

粒子（continuous）与网格（discrete）是 PIC 方法的两套坐标系统，算法的核心工作就是让信息在这两套系统之间来回插值。 Particles (continuous) and the grid (discrete) are two coordinate systems in PIC. The core task of the algorithm is to interpolate information back and forth between them. 粒子（連続）とグリッド（離散）は PIC 法の二つの座標系です。アルゴリズムの核心は、この二つの系の間で情報を相互に補間することです。

相同点 Similarities 共通点

GPU 和 CPU 本质上都是通用的数字电路处理器，都遵循冯·诺依曼架构：从内存读取指令和数据，执行运算，把结果写回内存。它们都包含 ALU、缓存，都可用 C/C++ 编程。 Both GPUs and CPUs are fundamentally general-purpose digital processors following the von Neumann architecture: fetch instructions and data from memory, execute computations, and write results back. Both contain ALUs, caches, and can be programmed in C/C++. GPU と CPU はどちらも根本的には汎用デジタルプロセッサであり、フォン・ノイマンアーキテクチャに従います：メモリから命令とデータを取得し、演算を実行し、結果をメモリに書き戻します。どちらも ALU とキャッシュを持ち、C/C++ でプログラム可能です。

与 PIC / GNN 的关系 Relation to PIC / GNN PIC / GNN との関係

EPOCH（PIC 代码）用 Fortran 编写，运行在 CPU 上——算法步骤有严格顺序依赖。

GNS（图神经网络）本质上是大量矩阵乘法和消息传递，天然适合 GPU（论文使用 Nvidia V100 / A100）加速。 EPOCH (PIC code) is written in Fortran and runs on CPU — its algorithmic steps have strict sequential dependencies.

GNS (graph neural network) is essentially large-scale matrix multiplications and message passing, naturally suited for GPU acceleration (Nvidia V100 / A100 in this paper). EPOCH（PIC コード）は Fortran で記述され、CPU 上で動作します。アルゴリズムのステップには厳格な順序依存関係があります。

GNS（グラフニューラルネットワーク）は本質的に大規模な行列積とメッセージパッシングであり、GPU（本論文では Nvidia V100 / A100）による加速に適しています。

什么是超参数 What is a Hyperparameter? ハイパーパラメータとは

超参数是指在模型训练开始之前就需要人为设定、而不是通过训练数据自动学习得到的参数。 A hyperparameter is a parameter that must be set by a human before training begins, rather than being learned automatically from training data. ハイパーパラメータとは、学習データから自動的に学習されるのではなく、学習開始前に人間が設定しなければならないパラメータのことです。

超参数控制训练过程本身，而非模型从数据中学到的内容。选错超参数，再好的数据也可能训练失败。 Hyperparameters govern the training process itself, not what the model learns from the data. Wrong hyperparameters can cause training to fail even with excellent data. ハイパーパラメータは学習プロセス自体を制御するものであり、モデルがデータから学ぶ内容ではありません。不適切なハイパーパラメータは、優れたデータがあっても学習の失敗を招く可能性があります。

4. GNS 神经网络示意图 4. GNS Neural Network Architecture 4. GNS ニューラルネットワーク構造図

论文中的图神经网络模拟器（Graph Network Simulator，GNS）采用经典的编码器—处理器—解码器（Encode-Process-Decode）结构。整个网络接收粒子和电场格点的历史状态作为输入，经过多轮消息传递学习粒子之间的相互作用，最终输出每个粒子的加速度和每个格点的电场预测值，从而推进模拟向前一个时间步。 The Graph Network Simulator (GNS) in this paper uses the classic Encode-Process-Decode architecture. The network takes the historical states of particles and electric field grid points as input, learns particle interactions through multiple rounds of message passing, and outputs predicted accelerations for each particle and electric field values for each grid point — advancing the simulation by one time step. 本論文のグラフネットワークシミュレータ（GNS）は、古典的なエンコード—プロセス—デコードアーキテクチャを採用しています。ネットワークは粒子と電場グリッド点の過去の状態を入力として受け取り、複数回のメッセージパッシングを通じて粒子間の相互作用を学習し、各粒子の加速度と各グリッド点の電場値を予測として出力し、シミュレーションを一タイムステップ進めます。

总体结构：Encode-Process-Decode Overall structure: Encode-Process-Decode 全体構造：エンコード—プロセス—デコード

整个网络分为三段。编码器把原始物理量翻译成网络可处理的潜在语言；处理器用 11 轮消息传递让节点之间充分交换信息，学习物理相互作用；解码器把最终的潜在向量直接映射到物理预测量。 The network has three stages. The encoder translates raw physical quantities into the latent language the network operates in. The processor uses 11 rounds of message passing to let nodes exchange information and learn physical interactions. The decoder maps the final latent vectors directly to physical predictions. ネットワークは三段階に分かれています。エンコーダは生の物理量をネットワークが処理できる潜在空間の表現に変換します。プロセッサは 11 ラウンドのメッセージパッシングによってノード間の情報交換を行い、物理的な相互作用を学習します。デコーダは最終的な潜在ベクトルを物理的な予測量に直接変換します。

消息传递的过程和 PIC 循环高度对应：场节点→粒子节点 ≈ 场插值到粒子；粒子节点→场节点 ≈ 电荷沉积；场节点间传递 ≈ 场求解（FDTD）。但与 PIC 不同，GNN 可以并行更新所有节点，且不受 CFL 条件限制。 The message-passing process closely mirrors the PIC loop: field→particle ≈ field interpolation; particle→field ≈ charge deposition; field↔field ≈ field solve (FDTD). Unlike PIC, however, the GNN can update all nodes in parallel and is not bound by the CFL condition. メッセージパッシングのプロセスは PIC ループに高度に対応しています：場→粒子 ≈ 場の補間；粒子→場 ≈ 電荷堆積；場↔場 ≈ 場の計算（FDTD）。ただし PIC とは異なり、GNN はすべてのノードを並列に更新でき、CFL 条件に縛られません。

编码器 Encoder エンコーダ

编码器是一个 MLP，对节点特征和边特征分别做嵌入，将其映射到 209 维潜在空间。隐藏层有 185 个神经元，只有 1 层（这是超参数搜索的结果，比原始默认的 2 层更优）。激活函数为 ReLU，输出端做层归一化（Layer Norm）。 The encoder is an MLP that embeds node and edge features separately, mapping them into a 209-dimensional latent space. It has 185 neurons in one hidden layer (found via hyperparameter search to outperform the default 2-layer version). The activation function is ReLU; the output is layer-normalised. エンコーダはノード特徴量とエッジ特徴量をそれぞれ埋め込む MLP であり、209 次元の潜在空間にマッピングします。隠れ層は 185 ニューロンの 1 層構成です（ハイパーパラメータ探索の結果、デフォルトの 2 層よりも優れていることが判明）。活性化関数は ReLU であり、出力には層正規化（Layer Norm）が適用されます。

处理器（消息传递） Processor — message passing プロセッサ — メッセージパッシング

处理器是网络的核心，执行 11 轮消息传递。每轮里，节点 \(i\) 的特征按如下公式更新： The processor is the core of the network, performing 11 rounds of message passing. In each round, node \(i\)'s features are updated as: プロセッサはネットワークの核心であり、11 ラウンドのメッセージパッシングを実行します。各ラウンドで、ノード \(i\) の特徴量は次のように更新されます：

\[\mathbf{x}_{i,m} = \mathbf{x}_{i,m-1} + F_m\!\left(\mathbf{x}_{i,m-1},\sum_{j}\phi_m(\mathbf{x}_{i,m-1},\mathbf{x}_{j,m-1},\mathbf{e}_{ij})\right)\]

其中 \(\phi_m\) 是计算"消息"的 MLP，\(F_m\) 是用消息更新节点特征的 MLP，两者在每轮都有独立的权重。消息先在所有邻居上求和（聚合），再和节点自身特征一起输入 \(F_m\)。每轮结束后对输出做层归一化。注意这里有一个残差连接（\(+\,\mathbf{x}_{i,m-1}\)），有助于梯度传播。 Here \(\phi_m\) is the MLP that computes "messages", and \(F_m\) is the MLP that updates node features using those messages — both have independent weights in each round. Messages are first summed over all neighbours (aggregation), then fed together with the node's own features into \(F_m\). Layer normalisation is applied after each round. Note the residual connection (\(+\,\mathbf{x}_{i,m-1}\)), which aids gradient flow. ここで \(\phi_m\) は「メッセージ」を計算する MLP、\(F_m\) はそのメッセージを用いてノード特徴量を更新する MLP であり、どちらも各ラウンドで独立した重みを持ちます。メッセージはまずすべての隣接ノードにわたって合算（集約）され、その後ノード自身の特徴量とともに \(F_m\) に入力されます。各ラウンド後に層正規化が適用されます。残差接続（\(+\,\mathbf{x}_{i,m-1}\)）により勾配の伝播が改善されます。

解码器与输出 Decoder & output デコーダと出力

解码器是一个没有隐藏层的 MLP（即线性映射），直接把 209 维潜在向量映射到物理预测量：粒子节点输出下一时间步的加速度，场节点输出电场值。注意解码器不做层归一化（论文特别指出，其他所有 MLP 的输出都做归一化，唯独解码器不做）。得到加速度后，用蛙跳格式积分更新粒子速度和位置，再构建新的图继续下一步 rollout。 The decoder is an MLP with no hidden layers (i.e. a linear map), directly mapping the 209-dimensional latent vector to physical predictions: particle nodes output acceleration for the next time step, field nodes output E-field values. Notably, the decoder does not apply layer normalisation (the paper specifically notes this is the only MLP whose output is not normalised). Acceleration is then integrated with leapfrog to update particle velocities and positions, and a new graph is constructed for the next rollout step. デコーダは隠れ層を持たない MLP（線形写像）であり、209 次元の潜在ベクトルを物理的な予測量に直接マッピングします：粒子ノードは次のタイムステップの加速度を、場ノードは電場値を出力します。注目すべき点として、デコーダは層正規化を適用しません（論文では、他のすべての MLP の出力は正規化されるが、デコーダだけが例外であると明記されています）。加速度は蛙跳び積分によって粒子の速度と位置の更新に用いられ、新しいグラフが構築されて次のロールアウトステップへと続きます。
